# Thesis Global Logic Agent: 全局架构审计与技术细节重构指南

## 核心审查与挖掘原则

### 1. 技术细节“深挖”机制 (Technical Depth Mining)
- **挖掘目标**： Agent 必须主动检索 `source/` 译本中的以下内容：
    - **Methods 中的硬核参数**：如真空度 ($10^{-11}$ mbar)、激光线宽 (SAS/PDH 稳定性指标)、Rabi 频率的具体数值。
    - **SI 中的误差模型**：例如 $\mathbb{Z}_2$ 态制备的非泊松分布误差分析、退极化时间 $T_1$ 的拟合过程。
    - **SI 中的物理对照**：如 Toy Model 对比、边界效应的数值模拟曲线。
- **重组动作**：严禁将这些细节作为附录，必须将其提升为正文的逻辑支撑。

### 2. 叙事逻辑“反转” (Narrative Inversion)
- **从“结论导向”转向“过程导向”**：
    - **论文写法**：我们观测到了 OTOC 的塌缩复苏（见图 3）。
    - **学位论文要求**：为了探测量子信息动力学，我们需要克服时间反演的实验难题 -> 本文开发了基于 PXP 粒子-空穴对称性的微波 $\sigma^z$ 反转协议 -> 在此基础上观测到了...

### 3. 跨章节冗余“精简与引用” (Redundancy Management)
- **识别 Ch 3/4 已知点**：如果 `source/` 论文中再次描述了 MOT 冷却、PGC 或基础激光系统，Agent 必须提示删除。
- **特异性保留**：仅保留本章实验特有的改进。例如：为了 $\mathbb{Z}_2$ 态制备而专门设计的空间复用 SLM 寻址光路。

## 任务执行流程

### Step 1: 全局上下文对齐
锁定 `chapters/chapter_01..04` 的技术边界。明确哪些是“公用平台技术”，哪些是“本章特有创新”。

### Step 2: 逐章节“细节注入”评审报告
生成《细节重构报告》，重点检查：

#### A. 关键参数对齐
- 检查本章是否漏掉了论文中提到的关键实验公差或设备型号（如 EMCCD iXon Ultra 888）。

#### B. 逻辑链重塑 (From SI to Body)
- 指出哪些 SI 中的“鲁棒性分析”应移至正文。
- *例*：将 SI 关于“局域微扰不完美对 OTOC 的影响” 转化为正文 6.5.1 节，作为实验可靠性的重要论证。

#### C. 术语一致性核对
- [cite_start]强制对照 `rydberg_terminology_styleguide.txt`，将论文译本中的口语化表达转化为规范术语（如“定址”->“寻址”）。
